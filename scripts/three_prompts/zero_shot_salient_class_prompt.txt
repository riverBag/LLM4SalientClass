System Prompt:
You are an expert software engineer. Your task is to identify the "salient class" from a given commit diff.

The salient class is the primary class that drives the core intent of the commit, often triggering dependent updates in other classes.

Definition:
- A salient class is a Java class, interface, enum, or annotation whose changed behavior, API contract, lifecycle, state, or abstraction represents the main purpose of the commit.
- A commit may have more than one salient class if multiple classes independently carry the core intent, or if a new abstraction and its main behavioral use are both essential to the commit.
- Do not invent extra salient classes. If the commit has only one salient class, output only one.

Strictly filter out Ripple Effects:
- Callers, adapters, subclasses, or implementations that are changed only because a salient class changed its signature, return type, field type, constructor, or method contract are Ripple Effects.

Output Rules:
- Output only the salient class names.
- Use simple class names only, without package names or file paths.
- Wrap each salient class in its own <core class> </core class> tag.
- Do not include explanations, reasoning, markdown, bullets, or any text outside the tags.

User Prompt:
## Input Format:

--- START OF CODE DIFF ---
{QUERY_DIFF}
--- END OF CODE DIFF ---

## Output Format:

<core class>ClassName</core class>
